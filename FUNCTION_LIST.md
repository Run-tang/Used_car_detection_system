# 车易检 - 二手车检测系统功能清单 (Functional Inventory)

本文档详细列出了车易检系统的功能模块、核心功能点，以及各业务板块在**列表页**与**详情/编辑页**中的关键字段定义。

---

## 1. 检测工单管理 (Work Order Management)
管理车辆检测任务的全生命周期，从创建到审核完成。

### 1.1 核心功能
- **工单创建**：支持手动录入或关联“库存车辆”自动填充。
- **状态追踪**：待派单、待检测、检测中、待审核、已完成、已取消。
- **派单分发**：手动指派或重新指派检测师。
- **详情查看**：集成车辆信息、检测师信息、检测进度及最终报告入口。

### 1.2 列表字段 (List Columns)
| 字段名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| 工单编号 | String | 系统自动生成的唯一业务标识 (如 WO-20231001-001) |
| 车辆信息 | Object | 显示：品牌车系、车牌号 |
| VIN | String | 车辆识别代号 |
| 检测师 | String | 当前指派人员姓名 |
| 任务状态 | Enum | 业务阶段标签 (Tags) |
| 预约时间 | DateTime | 计划检测时间 |
| 关联状态 | Boolean | 是否已关联车辆档案 |
| 操作区域 | Actions | 详情、派单、重派、预览报告 |

### 1.3 详情/表单字段 (Detail/Form Fields)
| 板块 | 字段名称 | 说明 |
| :--- | :--- | :--- |
| **基础信息** | 工单编号 | 只读 |
| | 预约时间 | 必填 |
| | 检测地点 | 经纬度及文本地址 |
| **车辆信息** | 品牌车系 | 必填 (如：奥迪 A4L) |
| | 车型年份 | 必填 (如：2023款) |
| | VIN 码 | 17位唯一码 |
| | 表显里程 | 单位：万公里 |
| | 车牌号 | 可选 |
| **保管/客户** | 联系人 | 车辆当前保管人姓名 |
| | 联系电话 | 必填 |
| **任务指派** | 指定检测师 | 下拉选择 (仅展示活跃检测师) |
| **备注信息** | 内部备注 | 多行文本，仅内部可见 |
| **关联档案** | 关联车辆 | 显示已关联的库存车辆 ID，支持跳转 |

---

## 2. 检测报告管理 (Inspection Report Management)
集中管理所有已生成的检测报告，并支持与车辆档案建立关联。

### 2.1 核心功能
- **全页预览**：沉浸式报告展示，包含评分抬头、异常汇总、分类导览及图文详情。
- **一车一档关联**：手动或自动关联检测结果至车辆档案。

### 2.2 列表字段 (List Columns)
| 字段名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| 报告编号 | String | 对应工单编号 |
| 车辆信息 | Object | 品牌型号、VIN 码 |
| 综合评分 | Number | 0-100 分 |
| 车况等级 | Enum | A/B/C/D 级 |
| 异常项数 | Number | 缺陷/异常统计 |
| 检测日期 | Date | 报告生成日期 |
| 操作区域 | Actions | 预览、导出PDF、关联档案 |

### 2.3 详情页内容 (Detail Page Content)
| 板块 | 内容项 | 说明 |
| :--- | :--- | :--- |
| **概览抬头** | 综合评分 | 大号数字展示 |
| | 车况等级 | 色块标签 (A绿/B蓝/C黄/D红) |
| | 报告总结 | 文本摘要，支持展开/收起 |
| | 车辆标签 | 自动生成的标签 (如：无事故、非水泡) |
| **异常汇总** | 缺陷列表 | 仅展示状态为“异常/Fail”的项目 |
| | 缺陷描述 | 问题描述及严重程度 (轻微/中度/严重) |
| **分类详情** | 左侧导航 | 证件、外观、机舱、底盘、内饰、路试等 |
| | 检测项 | 项目名称、是否必检 |
| | 结果状态 | 正常(Pass) / 异常(Fail) / 关注(Warning) |
| | 备注 | 检测师备注及审核员批注 |
| | 照片凭证 | 多张现场照片轮播预览 |

---

## 3. 检测师管理 (Inspector Management)
管理执行检测任务的人员信息及其工作效能。

### 3.1 核心功能
- **账户管理**：检测师基本信息编辑、账户状态切换。
- **效能分析**：实时统计关键 KPI，联动工单筛选。

### 3.2 列表字段 (List Columns)
| 字段名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| 姓名 | String | 检测师实名 |
| 城市/区域 | String | 负责业务区域 |
| 待检测数 | Number | <font color="blue">点击可跳转至待办工单</font> |
| 累计已检 | Number | 历史完成总量 |
| 平均耗时 | Number | 分钟/台 |
| 账户状态 | Boolean | switch 开关 (启用/禁用) |
| 操作区域 | Actions | 编辑、查看详情 |

### 3.3 详情/编辑字段 (Detail/Edit Fields)
| 板块 | 字段名称 | 说明 |
| :--- | :--- | :--- |
| **身份信息** | 头像 | 图片上传 |
| | 姓名 | 必填 |
| | 手机号 | 必填，作为登录账号 |
| | 负责城市 | 下拉选择 |
| **账户控制** | 账户状态 | 启用/禁用 (禁用后不可被派单) |
| **效能数据** | 当前任务 | 只读统计 |
| | 累计完成 | 只读统计 |
| | 平均耗时 | 只读统计 |

---

## 4. 检测模板管理 (Inspection Template Management)
根据不同车型配置个性化的检测项目集。

### 4.1 核心功能
- **模板定制**：配置检测大类、小项、是否必填等。
- **启停策略**：同一车型类型仅允许一个活跃模板。

### 4.2 列表字段 (List Columns)
| 字段名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| 模板名称 | String | 业务名称 |
| 适用车型 | Enum | 燃油 / 新能源 (Fuel/EV) |
| 包含大类 | Number | 一般为 8-10 个 |
| 包含小项 | Number | 细项总数 |
| 状态 | Enum | 已启用 / 草稿 / 停用 |
| 更新时间 | DateTime | 最后修改时间 |
| 操作区域 | Actions | 编辑、复制、预览、启用 |

### 4.3 编辑页字段 (Editor Fields)
| 板块 | 字段名称 | 说明 |
| :--- | :--- | :--- |
| **基础设置** | 模板名称 | 必填 |
| | 适用车型 | 单选 (燃油/新能源) |
| | 模板描述 | 选填 |
| **类目配置** | 大类名称 | 如：发动机舱检测 |
| | 排序权重 | 拖拽排序 |
| **检查项配置** | 项目名称 | 如：机油液位 |
| | 数据类型 | 通过/不通过、文本、数值、多选 |
| | 是否必填 | Boolean |
| | 拍照要求 | 最少照片数量限制 |
| | 检查指引 | 辅助说明文本 |

---

## 5. 库存车辆档案 (Vehicle Archives)
系统中的车辆身份基座，用于关联多次检测记录。

### 5.1 核心功能
- **档案建立**：车辆基础参数。
- **履历汇总**：查看该车辆的所有历史检测次数与报告分值曲线。

### 5.2 列表字段 (List Columns)
| 字段名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| VIN 码 | String | 核心索引键 |
| 品牌车型 | String | 品牌+车系+年款 |
| 车牌号 | String | 模糊搜索 |
| 里程 | Number | 万公里 |
| 状态 | Enum | 在库 / 已售 / 未知 |
| 建档时间 | DateTime | 首次录入时间 |
| 操作区域 | Actions | 查看详情、编辑 |

### 5.3 详情页字段 (Detail Fields)
| 板块 | 字段名称 | 说明 |
| :--- | :--- | :--- |
| **车辆参数** | VIN | 17 位码，不可修改 |
| | 品牌/车型 | 文本 |
| | 颜色 | 文本 |
| | 排放标准 | 国V / 国VI 等 |
| | 初登日期 | 日期选择 |
| **库存信息** | 存放仓库 | 文本地址 |
| | 保管人 | 姓名 |
| | 入库时间 | 日期 |
| **检测履历** | 历史报告 | 列表展示该车关联的所有报告 (时间/分数/等级) |
| | 分数趋势 | 折线图 (可选) |

---

## 6. 检测结果审核 (Inspection Audit)
质检环节，确保检测报告的真实性与准确性。

### 6.1 列表字段 (List Columns)
| 字段名称 | 类型 | 说明 |
| :--- | :--- | :--- |
| 工单号 | String | 关联 ID |
| 提交人 | String | 检测师 |
| 提交时间 | DateTime | |
| 车辆 | String | 简要信息 |
| 审核状态 | Enum | 待审核 / 已驳回 / 已通过 |
| 操作区域 | Actions | 审核 |

### 6.2 审核页字段 (Audit Page Fields)
| 板块 | 字段名称 | 说明 |
| :--- | :--- | :--- |
| **报告预览** | (嵌入) | 嵌入完整的检测报告只读视图 |
| **审核操作** | 审核结果 | 通过 / 驳回 |
| | 驳回原因 | 仅在“驳回”时必填 |
| | 关联档案 | Checkbox: 确认将此报告同步至车辆档案 |
| | 审核备注 | 内部留痕 |
